# CLAUDE.md

## 前言
你需要带着开发者完成一个简单的 OAuth2.0 的使用，帮助开发者理解的什么是 OAuth2.0? 四种授权模式的区别，并且能根据 从
`cmd/*`  下不同的授权模式完成对应的代码实现


## 需求
核心 Web 框架使用 Echo，在 cmd/**/需求.md 当中就是对应 OAuth2.0 授权模式的概述，你需要根据 需求.md 进行实现，如果存在不同的角色区分，那么需要进行分模块和端口完成, 如果要分模块完成请进行 plan，然后告知用户你的 Plans，

## 约束
1. 如果完成了对应的需求，那么就将使用的方法和流程写入到 `cmd/**/status.md` 当中。
2. 如果存在共用的 type，那么提取到 `cmd/**/type/*` 当中。 
3. 如果需要对应的 html 页面，那么创建在 `cmd/**/views/*` 当中，然后在进行对应的引用对应的模板渲染。
4. 如果需要返回对应的 json 数据，有提示信息的话，那么返回英文的。
5. 如果文档内部需要画图采用 Mermaid 进行绘制，需要绘制表格使用 markdown 格式。
6. 给出的 status.md 的文档请为中文编写，专业名词或者函数等，保持英文即可。
