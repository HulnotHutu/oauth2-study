OAuth2 Authorization Code + PKCE Flow

Components

  • Client Application - http://localhost:8080 (current)
  • Authorization Server - http://localhost:8081
  • Resource Server - http://localhost:8082

Status

  • Access Token: {{.TokenStatus}}
  • Refresh Token: {{.RefreshToken}}

Flow

  1. User clicks login → Client generates PKCE code_verifier + code_challenge (S256)
  2. Client redirects to Authorization Server with code_challenge (GET /authorize)
  3. User enters credentials and approves → Server stores code_challenge, redirects with ?code=xxx
  4. Client exchanges code + code_verifier for token (POST /token)
  5. Server verifies: SHA256(code_verifier) == code_challenge
  6. Returns access_token + refresh_token on success
  7. Client requests protected resource with token (GET /resource, Authorization: Bearer xxx)
  8. Resource Server introspects token (POST /introspect)
  9. When access token expires (401), client uses refresh_token to get a new access token
