OAuth2 Authorization Code Flow

Components

  • Client Application - http://localhost:8080 (current)
  • Authorization Server - http://localhost:8081
  • Resource Server - http://localhost:8082

Status

  • Access Token: {{.TokenStatus}}

Flow

  1. User clicks login → Client redirects to Authorization Server (GET /authorize)
  2. User enters credentials and approves → Authorization Server redirects back (302 ?code=xxx&state=xxx)
  3. Client exchanges code for token (POST /token)
  4. Client requests protected resource with token (GET /resource, Authorization: Bearer xxx)
  5. Resource Server introspects token (POST /introspect)
