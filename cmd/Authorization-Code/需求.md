1.3.1.  Authorization Code



&#x20;  The authorization code is obtained by using an authorization server

&#x20;  as an intermediary between the client and resource owner.  Instead of

&#x20;  requesting authorization directly from the resource owner, the client

&#x20;  directs the resource owner to an authorization server (via its

&#x20;  user-agent as defined in \[RFC2616]), which in turn directs the

&#x20;  resource owner back to the client with the authorization code.



&#x20;  Before directing the resource owner back to the client with the

&#x20;  authorization code, the authorization server authenticates the

&#x20;  resource owner and obtains authorization.  Because the resource owner

&#x20;  only authenticates with the authorization server, the resource

&#x20;  owner's credentials are never shared with the client.



&#x20;  The authorization code provides a few important security benefits,

&#x20;  such as the ability to authenticate the client, as well as the

&#x20;  transmission of the access token directly to the client without

&#x20;  passing it through the resource owner's user-agent and potentially

&#x20;  exposing it to others, including the resource owner.
----
"授权码是通过使用授权服务器作为客户端和资源所有者之间的中介来获得的。"

"由于资源所有者仅向授权服务器进行认证，因此资源所有者的凭证永远不会与客户端共享。"

"授权服务器再将资源所有者重定向回客户端并携带授权码。"

"将访问令牌直接传输给客户端，而无需经过资源所有者的用户代理。"

"授权码提供了一些重要的安全优势，例如能够对客户端进行身份验证。"
