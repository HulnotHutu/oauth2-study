OAuth2 Resource Owner Password Credentials Grant

Components

  • Client Application - http://localhost:8080 (current)
  • Authorization Server - http://localhost:8081
  • Resource Server - http://localhost:8082

Status

  • Access Token: {{.TokenStatus}}
  • Refresh Token: {{.RefreshToken}}

Flow

  1. User enters username and password on the client application
  2. Client sends POST /token with grant_type=password, username, password, client_id, client_secret
  3. Authorization Server validates credentials and client, returns access_token (+ optional refresh_token)
  4. Client uses access token to call Resource Server (GET /resource, Authorization: Bearer xxx)
  5. Resource Server introspects token (POST /introspect)
  6. When token expires, client can use refresh_token to obtain a new access token
