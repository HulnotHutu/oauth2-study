OAuth2 Implicit Grant Flow

Components

  • Client Application - http://localhost:8080 (current)
  • Authorization Server - http://localhost:8081
  • Resource Server - http://localhost:8082

Status

  • Access Token: {{.TokenStatus}}

Flow

  1. User clicks login → Client redirects to Authorization Server (GET /authorize, response_type=token)
  2. User enters credentials and approves → Authorization Server redirects back (302) with access_token in URI fragment
  3. Callback page JavaScript extracts token from fragment and stores it on the server
  4. Client requests protected resource with token (GET /resource, Authorization: Bearer xxx)
  5. Resource Server introspects token (POST /introspect)
