OAuth2 Client Credentials Grant

Components

  • Client Application - http://localhost:8080 (current)
  • Authorization Server - http://localhost:8081
  • Resource Server - http://localhost:8082

Status

  • Access Token: {{.TokenStatus}}
  • Client ID: {{.ClientID}}

Flow

  1. No user interaction required — the client authenticates itself
  2. Client sends POST /token with grant_type=client_credentials + Basic Auth header
  3. Authorization Server authenticates the client, issues access token (no refresh token)
  4. Client uses access token to call Resource Server (GET /resource, Authorization: Bearer xxx)
  5. Resource Server introspects token (POST /introspect)
  6. When token expires, client requests a new token (no refresh token available)
